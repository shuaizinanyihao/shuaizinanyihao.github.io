延迟任务实现方式汇总

发表于 2025-10-31 分类于 Java 热度： 3.9k 4 分钟

在现代业务系统中，延迟任务几乎无处不在：订单 30 分钟未支付自动取消、用户预约前提醒、缓存过期刷新、资源回收等。
传统方案如 Timer 或 ScheduledThreadPoolExecutor 虽然实现简单，但在高并发、大量任务场景下常会出现性能瓶颈和内存压力。

一、延时任务是什么？

延时任务（Delayed Task），指的是在指定延迟时间后或未来某个时刻执行的任务。
它是定时任务（Scheduled Task）的一个特例，更侧重于“延迟执行一次”而非“周期性执行”。

典型场景包括：

  • 电商：订单未支付超时自动关闭
  • 社交：消息撤回倒计时
  • 运维：延迟触发某项自动化任务

三、时间轮算法原理解析

时间轮（Time Wheel）是一种高效、低开销的延时任务调度模型，其核心思想是将时间“离散化”，用环形数组（类似时钟表盘）管理任务。

🔧 基本构成

  • 环形数组（Ticks）：类似钟表表盘，划分为固定格数（如 512 格），每一格是一个“时间槽（slot）”。
  • 指针（Cursor）：相当于秒针，每隔 tickDuration（如 100ms）移动一格。
  • 任务列表：每个 slot 存放需要在该时刻触发的任务链表。

🌀 工作机制

  1. 任务添加
    任务到达时，根据 delay 计算它应放入哪个 slot：
    slotIndex = (currentTime + delay) / tickDuration % wheelSize
  2. 指针轮转
    每隔 tickDuration，指针向前移动一格。
  3. 任务触发
    指针移动到的 slot 内所有任务被依次执行（通常提交到业务线程池，避免阻塞）。

⚡ 性能优势

  • 插入与删除任务复杂度均为 O(1)
  • 指针轮转成本固定，与任务数量无关
  • 在百万级任务场景下依然保持极高性能与低内存消耗
